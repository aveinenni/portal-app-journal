# portal-app-journal

本仓库用于归档和发布多个独立 HTML 页面。这些页面通常作为轻量级应用或信息展示页使用，不针对任何特定域名或网站。

## 项目简介

portal-app-journal 是一个静态页面集合仓库。您可以将多个独立的 HTML 文件放置于此，通过 GitHub Pages 或其他静态托管服务进行浏览和访问。适合存放工具页面、演示页面、存档页面等无需后端支持的纯前端内容。

## 目录说明

```
├── index.html          # 仓库入口页面（可选）
├── pages/              # 存放各独立 HTML 页面
│   ├── page1.html
│   ├── page2.html
│   └── ...
├── assets/             # 公共资源（CSS、JS、图片等）
│   ├── css/
│   ├── js/
│   └── images/
└── README.md           # 本文件
```

- `pages/`：主要存放归档的 HTML 页面文件，每个文件可独立访问。
- `assets/`：存放页面共享的样式、脚本或图片资源，便于统一管理。

## 页面归档说明

- 每个 HTML 页面应尽量保持独立，避免依赖外部不可控资源。
- 页面文件命名建议使用英文小写字母、数字和连字符，便于 URL 访问。
- 如需引用公共资源，请使用相对路径指向 `assets/` 目录。

## 维护说明

- 本仓库以归档和展示为目的，不提供在线编辑或动态功能。
- 欢迎通过 Pull Request 提交新的页面或更新已有页面。
- 提交前请确保页面内容合规、无恶意代码，并尽量保持风格简洁。
- 维护者会定期检查页面可用性，但不对页面内容承担额外责任。

## 使用方式

1. 克隆或下载本仓库到本地。
2. 在 `pages/` 目录下添加您的 HTML 文件。
3. 如需启用 GitHub Pages，请在仓库设置中开启，并选择 `main` 分支作为源。
4. 访问 `https://<用户名>.github.io/portal-app-journal/pages/your-page.html` 即可查看页面。
